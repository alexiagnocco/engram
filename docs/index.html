01 — the thesis

A knowledge base only compounds when retrieval outruns decay.

Three terms decide whether a memory system grows or rots. engram is engineered to push each one in the right direction.

σ

Coverage

How much of what you know you actually surface. Hybrid retrieval widens candidacy beyond keyword matches, so conceptually relevant notes still appear.

ρ

Precision

How useful what you surface turns out to be. A reinforcement-learning utility signal re-ranks toward notes that earn their place by being cited.

δ

Decay

How fast unused knowledge goes stale. Maintenance skills and health metrics keep the graph linked and reachable, holding decay down.

When  σ·ρ  exceeds  δ/100, the system is above escape velocity — it accumulates useful, retrievable knowledge faster than it loses it.

This compounding model is adapted from AgentOps · The Science by boshu2 (Apache-2.0) — see credits.

02 — the engineering centerpiece

A two-stage hybrid ranker with a learned utility signal.

The retrieval tool, vault_retrieve, scores every note two ways, fuses them, then re-ranks the top pool by a reinforcement-learning reward that the system teaches itself.

  1. 1

    Fusion & candidacy

    Every eligible note is scored by a keyword composite and dense-vector cosine similarity at once, then ranked on the normalized sum. Candidacy no longer needs a keyword hit — a conceptually relevant note with zero shared words still surfaces.

    score = zNorm(match·3 + freshness·2 + connectivity·1)
      + W_DENSE · zNorm(cosine(q, note))
  2. 2

    MemRL re-ranking

    The top pool is re-ranked by a learned utility signal. When a retrieved note is later cited in the work, its utility is rewarded; when it's surfaced but ignored, it isn't — tracked as an exponential moving average per note.

    rank += LAMBDA_UTILITY · zNorm(utility)
utility ← (1 − α)·utility + α·reward      # α = 0.3

The compounding loop

  1. retrieve hybrid-ranked notes
  2. work cites what's useful
  3. feedback rewards cited notes
  4. re-rank sharpens next time

Retrieval feeds work; work emits a signal; the signal sharpens the next retrieval. That's the loop that makes the base compound instead of just grow.

Measured, not assumed

vault_feedback records the reward · vault_sigma_rho reads accumulated feedback back out as measured coverage/precision · vault_health reports whether the whole system is above escape velocity.

Pluggable embeddings

A dependency-free hashing backend (deterministic SHA-1 feature hashing, always available) or a semantic ONNX backend (a real sentence-transformer via onnxruntime). Vectors cache by content hash — only changed notes re-embed.

05 — quick start

Filesystem-first. No external services required.

Python 3.12 + uv. Register once; the 23 tools load automatically in any MCP client.

A dependency-free hashing backend works out of the box. Drop the ONNX model in for real semantic retrieval. Live two-way sync with Obsidian is optional via the Local REST API.

06 — measured quality

Skills are tuned against graded benchmarks, not vibes.

A runnable harness scores the /execute skill across three scenarios — ambiguous scope, an over-engineering bait, and a security refactor — with and without the skill loaded. Before/after grades make a regression in skill quality measurable instead of anecdotal.

07 — credits & prior art

Built on AgentOps' science.

engram's knowledge-compounding model — the σ·ρ > δ/100 escape-velocity thesis, the σ/ρ/δ terms, the MemRL utility loop, the 40% context rule — is adapted from AgentOps · The Science by boshu2 (Apache-2.0). engram is an independent implementation — it includes no AgentOps code.

  • AgentOps · The Science — the compounding model, escape velocity, σ/ρ/δ, and the MemRL framing. the-science · repository
  • Ebbinghaus (1885) — Memory: A Contribution to Experimental Psychology. The forgetting curve; the basis for treating knowledge as decaying (δ).
  • Darr, Argote & Epple (1995) — "The Acquisition, Transfer, and Depreciation of Knowledge in Service Organizations," Management Science. Source of the ~17%/week decay rate.
  • Liu et al. (2023) — "Lost in the Middle: How Language Models Use Long Contexts." arXiv:2307.03172. Long-context utilization — the 40% rule.
  • MemRL (2026) — "Self-Evolving Agents via Runtime Reinforcement Learning on Episodic Memory." arXiv:2601.03192. Two-phase retrieval (semantic filter → learned utility) — engram's fusion → MemRL re-rank mirrors it.
